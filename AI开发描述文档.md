# AI开发描述文档

这个文档用于约定：以后如果你要让我新增一个公共组件，应该怎么描述需求，才能让我一次性把组件实现、导出、README、manifest 和统一入口都补齐。

## 推荐原则

- 不需要把每个文件都点名告诉我
- 重点告诉我组件名、所属包、解决的问题、页面希望怎么用
- 如果没有特殊说明，我默认按当前组件库规范自动补齐文档和导出
- 如果你没有要求，我默认不提交、不发布

## 最短版描述模板

```text
帮我新增一个公共组件，组件名叫 {{组件名}}，放到 {{ui-core / ui-energy}}。
这个组件用来解决 {{一句话说明用途}}。
页面侧希望这样用：{{怎么传参、怎么调用}}。
请按当前组件库规范完成实现、导出、README、manifest、统一入口接入，但先不要提交和发布。
```

## 通用完整版描述模板

```text
帮我新增一个公共组件。

组件信息：
- 组件名：{{组件名}}
- 所属包：{{@hbdlzy/ui-core / @hbdlzy/ui-energy}}
- 用途：{{这个组件解决什么问题}}
- 适用场景：{{哪些页面/功能会用}}
- 页面用法：{{页面里怎么引入、传什么 props、是否需要 expose 方法}}
- 是否需要统一从 @hbdlzy/ui 导出：{{是/否}}
- 是否需要默认样式或主题变量：{{是/否，说明一下}}
- 是否需要工具函数一起导出：{{是/否，说明一下}}
- 是否需要补示例：{{是/否}}
- 要求：请按当前组件库规范自动补齐实现、index.ts、README、manifest、必要文档入口，但先不要提交和发布。
```

## 最少需要告诉 AI 的 4 个信息

如果你想尽量少写，至少告诉我：

- 组件名
- 放 `ui-core` 还是 `ui-energy`
- 这个组件解决什么问题
- 页面希望怎么用

只要这 4 个信息明确，我就可以按现在的组件库规范继续落地。

## AI 默认会自动处理的内容

如果你没有特殊说明，我会默认帮你处理：

- 组件源码目录
- 组件 `index.ts`
- 包级 `src/index.ts`
- 组件 `README.md`
- 包级 `manifest`
- 如果需要统一入口，再接入 `@hbdlzy/ui`
- 必要时更新总文档入口
- 构建验证

## 示例

```text
帮我新增一个公共组件，名字叫 BaseUpload，放到 ui-core。它用来统一文件上传入口，页面里只想传 uploadUrl、fileList、limit 这些配置，并能拿到上传成功后的回调。请按当前组件库规范把实现、导出、README、manifest 都补齐，先不要提交和发布。
```

## 补充约定

- 如果组件属于基础能力，优先放 `ui-core`
- 如果组件带储能业务语义，优先放 `ui-energy`
- 如果页面只是偶尔使用某个工具函数，也可以直接说明“需要工具函数一起导出”
- 如果你希望我保留更大自主性，可以直接说“按当前组件库规范自行判断并落地”
